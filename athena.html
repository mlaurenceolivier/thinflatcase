Athena CMS

A Content Management System for the PNR Ecosystem
Athena Content Management System
"It needs to function the same...but better."

Role: Lead Designer | When: 2018 | Tools: Sketch/Invision App

After working on the Pick-n-Roll Style Guide with another designer, we decided to test out the new PnR Style Guide’s applicability to a new product. Enter Athena CMS.

Originally built to power Huffington Post’s content contributors, Athena was going to move under the Pick-n-Roll suite and would be used by all content contributors. We needed to keep most of the overall functionality with key features specifically for either legacy Aol contributors or Yahoo contributors.

We couldn’t stick with the original UI as it was purely branded as Huffington Post so we felt this would be a good testing ground to apply the PnR Style Guide and see if we could break it.

WHAT’S THE GOAL HERE?

At first, I needed to run through what elements and features would be retained and how I could best organize those elements and features to make for an easier to use, cleaner UI. I worked with the existing PM on the project to determine what would stay and what would go.

Once I had a solid list of MVPs for the first iteration, I created a new Sketch file, linked up the library I had created from our PnR Style Guide and got to work.

THE SAME, BUT DIFFERENT

The first problem I ran into was that PnR was built on a one-column layout system. I needed to use the same basic 12-column grid (minus the left-nav) and generate a concise way to handle 3-columns of content. The easiest method was to simply add the main content area in the middle 6 columns and then one each of 3 columns on either side.

One way I decided to divide the UI into 3 parts was to:

  • Content specific to the creator would be on the left (creator’s name, who’s currently editing the content and internal editor comments)
  • Content creation all took place in the main column
  • Meta-level content (URL, preview image, external credits, etc.) would be on the right along with the Publish and Schedule Post buttons
  • This layout worked well in testing but required a new set of form fields, buttons and elements to fit in the new 3- and 6-column widths. I got to work

With all of the new elements created as symbols, I was easily able to iterate some various mocks of the most basic types of layouts for the new product.

Now that the basic UI was in place and workable, the focus moved to the UX improvements.

Not being familiar with the original product helped me find the gaps in their current work flow and easily determine improvements to the UX. There were quite a few simple things that began to jump out:

  • Creating and editing content blocks with one click
  • Editing existing content by clicking into the content instead of clicking icons/menus
  • Moving blocks of content around with a simple drag-n-drop function instead of clicking icons, opening modals, etc.
  • Deleting content blocks with a simple icon instead of navigating menus

The new Athena product has been shipped to engineers for development and has soft launched to a select group of power users.
